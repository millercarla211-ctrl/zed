# Flow Competitive Scorecard

Date measured: **April 26, 2026**

This is an internal product-completion score for Flow as it exists in this repository. It is not a market-share score and not a benchmark of model quality. It is a weighted completion score against three targets:

1. Wispr Flow parity
2. Grammarly parity
3. Flow-native local/runtime advantages

## Current Scores

- **Overall Flow competitive completion:** `51 / 100`
- **Wispr Flow replacement readiness:** `52 / 100`
- **Grammarly replacement readiness:** `40 / 100`
- **Flow-native local/runtime advantage readiness:** `57 / 100`

## Biggest Gaps To Reach 100

These are the highest-value gaps:

1. Finish polished system-wide product surfaces across desktop and mobile.
2. Ship real IDE variable recognition and live file tagging integrations.
3. Add plagiarism, citation, fact-checking, and academic-assistance features.
4. Connect the provider layer to real local/remote auto-switching.
5. Finish multimodal local execution and conversion/publishing pipelines.

## Practical Interpretation

Flow is already ahead of Wispr Flow and Grammarly in one important direction:

- local-first runtime architecture
- offline model execution
- embeddable Rust library design
- multi-project DX integration
- planned local/remote unified routing

But Flow is still behind both products in product polish and end-user completeness.

That is why the current score is **51 / 100** instead of something inflated.

## Near-Term Goal

The next credible milestone is not 100 immediately.

The next credible milestone is:

- **65 / 100 overall**
- **70 / 100 Wispr parity**
- **55 / 100 Grammarly parity**

That would mean Flow has crossed from architecture-heavy to genuinely competitive product behavior.
